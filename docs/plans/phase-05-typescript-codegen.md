# Phase 05 — Go → TypeScript codegen + drift cross-check

## Summary

Phase 05 completes the Design A codegen pipeline (RFC §6.2): it generates the
TypeScript half of a tool contract — `web/src/generated/contracts.ts` — from the
same Go contract structs that Phase 04 turns into JSON Schema, using the pure-Go
AST generator `github.com/gzuidhof/tygo`. It also ships the **drift cross-check**:
a library function — the seam Phase 18's `dockyard validate` will call — that
hard-fails when the generated JSON Schema and the generated TypeScript desync, or
when generated output on disk is stale versus its Go source.

## RFC anchor

- RFC §6.2 — the codegen pipeline (Design A): schema and TypeScript are generated
  independently from Go, each by a pure-Go tool, and `dockyard validate`
  cross-verifies schema vs. TS and hard-fails on drift or stale output.
- RFC §6.1 — the Go contract struct is the single source of truth (P1).
- RFC §4.3 — the generated project layout names `web/src/generated/contracts.ts`.

## Briefs informing this phase

- brief 06 — Go-2026 no-CGo stack & toolchain.
- brief 04 — mcp-use DX teardown.

## Brief findings incorporated

- **brief 06 §2.4 / §3.1 (Design A).** "`gzuidhof/tygo` is the strongest option:
  it generates TypeScript from Go **source** (AST parsing, not reflection), so it
  preserves doc comments, understands constants/enums and generic types." Phase 05
  adopts tygo and drives it through `tygo.ConvertGoToTypescript`, which converts
  Go contract source straight to TypeScript with no Node dependency anywhere on
  the codegen path.
- **brief 06 §3.1 / R1 (the drift check is non-optional).** "Generating schema and
  TS from Go independently means a bug in one generator silently desyncs them …
  `dockyard validate` must cross-verify." Phase 05 ships that cross-verifier as a
  library function (`codegen.CrossCheck`) so Phase 18's CLI is a thin caller.
- **brief 06 R1 (stale generated output is a build blocker).** "the braindump's
  'generated types out of date = build blocker.'" Phase 05's `codegen.CheckStale`
  compares on-disk generated bytes against a fresh regeneration and fails on any
  difference — staleness is an error, never a warning.
- **brief 04 §2.6 (the defect Dockyard closes).** mcp-use's widget types are
  hand-declared with nothing tying them back to the tool's output schema, so
  server↔UI drift is silent. The cross-check is the mechanism that makes that
  drift loud in Dockyard.
- **brief 06 R4 (keep contract structs simple; verify tygo's generic handling).**
  Phase 05 keeps the generated TS boring and readable, deterministic-marshalled
  with a `Code generated … DO NOT EDIT.` header, and golden-tested.

## Findings I'm departing from (if any)

None. Phase 05 implements Design A as the RFC and brief 06 settle it.

## Goals

- Generate compilable, deterministic TypeScript from Go contract structs.
- Generated TS carries a `Code generated … DO NOT EDIT.` header and is pinned by
  golden tests.
- Ship a drift cross-check library: schema↔TS desync detection and stale-output
  detection, both hard-failing, exposed as a clean callable seam for Phase 18.
- The whole codegen path stays pure-Go — no Node dependency, no CGo.

## Non-goals

- The `dockyard generate` and `dockyard validate` **CLI commands** — Phase 18.
  Phase 05 ships the library; the CLI wires it.
- Watching contract files and regenerating on save — that is `dockyard dev`
  (Phase 17), which calls this package.
- Manifest-driven resolution of a Go type reference to a package — Phase 06 owns
  the manifest; Phase 18 wires manifest → codegen.
- The `content` vs `structuredContent` validate warning (RFC §6.3) — Phase 08 /
  Phase 18.

## Acceptance criteria

- [ ] A Go contract struct generates correct, compilable TypeScript
      (`web/src/generated/contracts.ts`-shaped output) via `tygo`.
- [ ] Generated TypeScript is deterministic (identical input → byte-identical
      output) and carries the `Code generated … DO NOT EDIT.` header.
- [ ] The drift cross-check detects a schema↔TS desync (a property present in one
      artifact and absent or differently-optional in the other) and fails.
- [ ] The drift cross-check detects stale generated output (on-disk bytes differ
      from a fresh regeneration) and fails.
- [ ] Golden tests cover the generated TypeScript output.
- [ ] The codegen path is pure-Go and builds with `CGO_ENABLED=0`.

## Public API surface

```go
// internal/codegen

// TypeScriptForSource converts Go contract type-declaration source into
// deterministic TypeScript with a Code-generated header.
func TypeScriptForSource(goSource string, opts ...TSOption) ([]byte, error)

// TypeScriptForDir reads the .go files of a contracts directory and generates
// TypeScript for the type declarations they contain. This is the seam Phase 18's
// `dockyard generate` calls.
func TypeScriptForDir(dir string, opts ...TSOption) ([]byte, error)

// TSOption configures TypeScript generation (indentation, optional-type style).
type TSOption func(*tsConfig)
func WithIndent(indent string) TSOption
func WithNullOptional() TSOption

// CrossCheck verifies the generated JSON Schema for a contract and the generated
// TypeScript interface for the same contract describe the same property set with
// consistent optionality. It returns an error wrapping ErrSchemaTSDrift on any
// desync. The seam Phase 18's `dockyard validate` calls.
func CrossCheck(schema *jsonschema.Schema, tsTypeName string, ts []byte) error

// CheckStale reports whether on-disk generated output is stale versus a fresh
// regeneration. It returns an error wrapping ErrStaleGenerated on any difference.
func CheckStale(onDisk, fresh []byte) error

var ErrSchemaTSDrift  = errors.New(...)
var ErrStaleGenerated = errors.New(...)
```

## Test plan

- **Unit:** `TypeScriptForSource` on scalars, nested structs, slices, maps,
  pointers, enums, optional fields; determinism (same input twice → identical
  bytes); header present; `WithNullOptional` shape. `TypeScriptForDir` on a
  multi-file fixture directory. `CrossCheck` — passing case and each desync class
  (missing property, extra property, optionality mismatch, non-object schema,
  missing TS interface). `CheckStale` — identical vs. drifted bytes.
- **Integration:** `test/integration/phase05_drift_test.go` — a Go contract pair
  drives `SchemaFor` (Phase 04) + `TypeScriptForSource` (Phase 05); `CrossCheck`
  passes on the matched pair and fails on a mutated TS file; `CheckStale` flags a
  stale fixture. Exercises the Phase 04 → Phase 05 seam with no mocks.
- **Concurrency / golden:** golden tests pin the generated TS
  (`testdata/*.ts.golden`, regenerated with `-update`). `TypeScriptForSource` and
  the drift checks are pure functions with no shared state — a parallel
  `t.Parallel()` golden run proves concurrent-safe reuse.

## Risks / open questions

- **R1 — tygo source parsing (brief 06 R4).** `tygo.ConvertGoToTypescript` parses
  a Go type-declaration fragment; a contract using an unsupported construct
  surfaces as a tygo parse/convert error wrapped by Phase 05. Mitigation: keep
  contract structs simple (boring generated code is the mandate anyway) and
  golden-test the supported shapes.
- **R2 — cross-check fidelity.** `CrossCheck` compares property name sets and
  optionality, not full structural type equality — a same-named property with a
  divergent value type is not caught structurally. This is bounded: both
  artifacts derive from the same Go field, so a type divergence would be a tygo
  or jsonschema-go bug, and golden tests pin each generator independently. Full
  type-graph equality is a possible later hardening (noted in D-034).
- **R3 — RFC §18 Q-6.** The `google/jsonschema-go` version is kept in lockstep
  with the SDK (D-030); unchanged by this phase.

## Glossary additions

- **drift cross-check** — the `internal/codegen` library check that fails when the
  generated JSON Schema and the generated TypeScript for a contract desync, or
  when generated output is stale versus its Go source. Phase 18's `dockyard
  validate` calls it. RFC §6.2. D-034.
- **generated TypeScript** — `web/src/generated/contracts.ts`: the TypeScript
  types generated from the Go contract structs by `tygo`; the UI-facing half of
  the Design A pipeline. Never hand-authored. RFC §6.2. D-032.
- **stale generated output** — a generated artifact whose on-disk content no
  longer matches a fresh regeneration from its Go source. A build blocker, never
  a warning. RFC §6.2. D-034.
