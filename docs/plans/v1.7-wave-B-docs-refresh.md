## Summary

The docs site (`docs/site/`, published to GitHub Pages) lags the code by seven
releases. The auto-generated and transcluded surfaces are fine; the hand-written
prose and code sketches are not. This wave refreshes them to v1.7.x and corrects
the inaccuracies the audit found, the largest being **every Svelte App sketch
shows a dead `dockyard-bridge` API**.

Headline defects:

1. **The Svelte sketches use a `dockyard-bridge` API that does not exist.** They
   show `import { hostContext, onToolResult } from 'dockyard-bridge'` and a
   top-level `onToolResult((p) => â€¦)`. The real package exposes a `createBridge()`
   factory â†’ `bridge.onToolResult<S>((r) => r.structuredContent)`,
   `bridge.onHostContextChanged((p) => p.styles?.variables)`, `bridge.connect()`,
   and reactive `bridge.hostContext` stores. A reader copy-pasting the docs gets
   code that does not compile. (`getting-started/analytics-widgets.md`,
   `guides/ui-resources.md`.) Source of truth:
   `templates/analytics-widgets/web/src/App.svelte`.
2. **Stale version pins.** `## Released: v1.0.0` and
   `go install â€¦@v1.0.0` on the home page and getting-started; "post-v1.0.0",
   "V1 templates", "V1 hosts" phrasing.
3. **The CLI reference lists a non-existent `inspector` template.** Root cause is
   upstream: `internal/cli/new.go`'s `dockyard new` long-description hardcodes
   "Templates (analytics-widgets, approval-flows, inspector)". The
   auto-generated `cli/index.md` faithfully copies it. Fix the source, then
   `make docs` regenerates â€” editing the page directly would be reverted.
4. **Â§19 contributor-vocabulary leaks.** A prose leak ("the phase-25 layout fix"
   in `approval-flows.md`) and `/screenshots/phase-NN/` image paths across
   several pages.
5. **Coverage gaps.** The `dockyard-ui` npm package is undocumented; the
   inspector guide does not mention it is now the validating, content-sizing
   host (the v1.7 work); the approval-flows page does not flag that inline
   elicitation is a Dockyard-host-only extension (D-183); the excellent
   ui-resources handshake-troubleshooting section is not cross-linked from the
   pages where a reader hits the same symptoms.

### What is NOT broken (audit confirmed drift-proof)

- **`cli/index.md` is auto-generated** by `internal/clidocs` via `make docs`,
  regenerated in CI (`.github/workflows/docs.yml`) â€” byte-identical to a fresh

- **`reference/*.md` are `@include` transclusions** of the canonical in-repo

- **`agent-skills/index.md`** lists all 8 skills correctly.

## Risks / open questions

- **Screenshot-path rename (the one bigger mechanical change).** The
  `/screenshots/phase-24-finish/` and `/screenshots/phase-25/` image paths leak
  methodology vocabulary on user-facing pages. `drift-audit` tolerates lowercase
  `phase-NN` *paths* (it treats them as filesystem metadata, not prose), so this
  is hygiene, not a CI failure. Renaming the public screenshot dirs to
  versionless names (e.g. `analytics-widgets/`, `approval-flows/`) touches
  `docs/screenshots/**` + every referencing page. **Recommendation:** do the
  cheap prose leak now (item 4) and decide rename-vs-defer with the maintainer;
  it can ride this wave or a follow-up. (Open question for the maintainer.)
- **Docs build is a standalone workflow.** `make docs` is gated in
  `docs.yml`, not `ci.yml`. Worth confirming the docs build is a *required*
  status check so a docs break blocks merge (a CI-config nit, optionally folded
  in).
- **Decision entry vs none.** A docs refresh + a one-line CLI-copy fix may not
  warrant a `D-NNN`; judge at implementation (the precedent files decisions
  liberally, but a copy fix is borderline).
