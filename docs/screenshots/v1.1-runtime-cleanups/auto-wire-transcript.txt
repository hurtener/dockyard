STEP 7 — tasks/list correctly withheld (RequestorIdentifiable: false default)
-----------------------------------------------------------------------------
$ curl -s -X POST http://127.0.0.1:8765/ \
    -H "Content-Type: application/json" \
    -H "Accept: application/json, text/event-stream" \
    -d '{"jsonrpc":"2.0","id":2,"method":"tasks/list","params":{}}'

{"jsonrpc":"2.0","id":2,"error":{"code":-32601,"message":"dockyard/runtime/tasks: unknown tasks method: \"tasks/list\" (tasks/list not advertised)"}}

  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
  The auto-wire defaults to RequestorIdentifiable: false (the single-user
  stdio posture — brief 02 §4.5). tasks/list is correctly withheld with
  the spec's -32601 method-not-found rather than advertised against a
  receiver that cannot identify requestors. To enable list-in-HTTP, edit
  main.go's tasks.Options to set RequestorIdentifiable: true and pass an
  AuthContextFunc via server.Options.TasksAuthContext.
