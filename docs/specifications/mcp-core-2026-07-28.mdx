# MCP core draft 2026-07-28

This pinned draft defines the modern MCP lifecycle Dockyard targets in the V2
migration wave. It is a release-candidate snapshot, not a final-conformance
claim.

## Consumed requirements

- Every modern request carries protocol version, client identity, and client
  capabilities in request `_meta`; HTTP mirrors the version in
  `MCP-Protocol-Version`.
- The modern HTTP lifecycle is POST-only and stateless: no protocol session,
  no `Mcp-Session-Id`, and no standalone GET SSE stream.
- `server/discover` reports supported versions, capabilities, and server
  identity before other calls.
- A dual-era server chooses the lifecycle from declared protocol version. It
  never silently downgrades or uses a JSON-RPC body heuristic.

The complete upstream snapshot is identified by the header pin. Phase 32 owns
the runtime implementation; Phase 34 owns routing-header and schema semantics.
