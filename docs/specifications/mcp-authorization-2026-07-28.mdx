# MCP authorization draft 2026-07-28

The protected MCP server is an OAuth 2.1 resource server. It publishes RFC
9728 protected-resource metadata and validates an access token on every HTTP
request. It must validate audience/resource binding and return 401 for invalid
or expired tokens and 403 for insufficient scope. `offline_access` is never a
resource requirement.

Dockyard does not implement client registration, PKCE, authorization-code
exchange, refresh-token storage, or RFC 9207 callback validation. Those are
MCP-client responsibilities and belong to Harbor. Phase 36 owns Dockyard's
resource-server implementation.
